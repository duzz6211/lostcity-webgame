# RULES.md — Claude 작업 규칙 (최상위)

> 이 문서는 **모든 규칙의 최상위**다. `TASK.md`를 포함한 다른 어떤 지시보다 우선한다.

---

## 0. 우선순위 (Precedence)

- 우선순위: **RULES > TASK > 기타 지시**.
- RULES와 충돌하는 지시(TASK.md 내용 포함)는 **따르지 말고 먼저 사용자에게 확인**한다.
- RULES에 명시되지 않았다고 임의로 해석·확장하지 않는다. 애매하면 묻는다.

---

## 1. 실행 환경 (Environment)

- 환경은 **Windows PowerShell이 아니다.** → **Windows → WSL → Ubuntu (bash)** 환경이다.
- 모든 명령·스크립트·경로는 **Ubuntu(bash) 기준**으로 작성한다.
  - PowerShell / CMD 문법 사용 금지 (예: `;` 체이닝, `Set-Location`, 백슬래시 경로 등 금지).
  - 경로는 WSL/리눅스 경로 기준(`/home/...`, `/mnt/c/...`). Windows 경로(`C:\...`) 가정 금지.
- 명령 예시를 적을 때도 bash 문법으로만 제시한다.

---

## 2. 에이전트가 허락 없이 해도 되는 점검 (Allowed checks)

자율적으로 수행해도 되는 **점검은 아래 네 가지뿐이다.**

1. **수치 재확인** — 문서·코드 내 숫자/값의 정확성 재검토
2. **일치 검토** — 슬라이드/섹션/파일 간 내용 일관성 검토
3. **오탈자 점검**
4. **정의 누락 점검** — 용어·항목·참조가 정의 없이 사용됐는지 확인

> 위 네 가지 외의 다른 점검·검사·분석·"개선 제안 실행"은 **하지 않는다.**
> 발견한 다른 이슈는 **실행하지 말고 보고만** 하고, 사용자의 추가 허락을 기다린다.

---

## 3. 허락 전까지 절대 금지 (Forbidden until explicitly permitted)

- **시키지 않은 명령 실행 금지.** 특히 다음 같은 실행성 작업을 임의로 돌리지 않는다:
  - `npm run dev`, 개발 서버 구동, `build`, `test`, `install`, 배포, 기타 프로세스 실행
  - → 이런 **실행은 사용자가 직접 한다.** 에이전트는 코드·파일의 **작성/수정까지만** 한다.
- **2번에 없는 추가 점검·검사**를 임의로 수행하지 않는다.
- **파괴적 작업 금지**: 파일 삭제, 대규모 일괄 변경, `git push`/커밋, 의존성 변경 등은 먼저 묻는다.
- 요청 범위를 벗어나는 리팩토링·구조 변경·라이브러리 추가를 임의로 하지 않는다.

---

## 4. 작업 방식 (How to work)

- 요청한 범위만 정확히 수행한다. 범위를 넘는 일이 필요해 보이면 **실행하지 말고 먼저 묻는다.**
- 확신이 없으면 추측으로 진행하지 말고 확인한다.
- 무언가를 멈추고 사용자 결정을 기다려야 할 때는, **무엇을/왜** 멈췄는지 짧게 보고한다.
