# Quoridor 웹 게임 구현 명세서

> 이 문서는 Claude Code(CLI)가 Quoridor 보드게임을 웹 기반으로 구현할 수 있도록 작성된 기술 명세서입니다.

---

## 1. 게임 개요

- **장르**: 추상 전략 보드게임 (2인 또는 4인)
- **목표**: 자신의 말(pawn)을 반대편 끝줄(goal row)에 가장 먼저 도달시키기
- **핵심 메커니즘**: 매 턴 "말 이동" 또는 "벽 설치" 중 하나를 선택

---

## 2. 보드 구조

### 2.1 셀(Cell) 그리드

- 보드는 **9 × 9 = 81 칸**의 정사각형 그리드
- 좌표계: `(col, row)`, 0-indexed, `col ∈ [0..8]`, `row ∈ [0..8]`
- `row = 0`이 상단, `row = 8`이 하단 (또는 반대로 일관되게)

### 2.2 벽(Wall) 슬롯

- 벽은 **칸 사이의 홈(slot)** 에 설치됨
- 한 벽은 항상 **인접한 두 슬롯(2칸 길이)** 을 점유
- 가능한 벽 방향: **수평(horizontal)** 또는 **수직(vertical)**
- 벽 좌표계: `(col, row, orientation)`
  - `col ∈ [0..7]`, `row ∈ [0..7]` (8 × 8 = 64개의 교차점)
  - `orientation ∈ {"H", "V"}`
- **수평 벽** `(c, r, "H")`: 셀 `(c, r)`-`(c, r+1)` 사이 + `(c+1, r)`-`(c+1, r+1)` 사이를 막음
- **수직 벽** `(c, r, "V")`: 셀 `(c, r)`-`(c+1, r)` 사이 + `(c, r+1)`-`(c+1, r+1)` 사이를 막음

## 4. 게임 상태 (Game State) 데이터 모델

```typescript
type Orientation = "H" | "V";

interface Wall {
  col: number;          // 0..7
  row: number;          // 0..7
  orientation: Orientation;
}

interface Player {
  id: "P1" | "P2" | "P3" | "P4";
  position: { col: number; row: number };
  wallsRemaining: number;
  goal: (pos: {col:number,row:number}) => boolean; // 도달 판정 함수
}

interface GameState {
  players: Player[];
  currentTurn: number;          // players 배열의 인덱스
  walls: Wall[];                // 보드 위에 놓인 모든 벽
  history: Move[];              // 기보(replay/undo용)
  winner: Player["id"] | null;
  status: "playing" | "finished";
}

type Move =
  | { type: "MOVE"; to: { col: number; row: number } }
  | { type: "WALL"; wall: Wall };
```

## 5. 턴 진행 규칙

1. 각 플레이어는 자기 턴에 **다음 중 정확히 하나**를 수행:
   - **(A) 말 이동 (MOVE)**
   - **(B) 벽 설치 (WALL)**
2. 행동이 끝나면 다음 플레이어로 턴이 넘어감.
3. **패스(pass)는 불가능**. 가능한 행동이 하나라도 있으면 반드시 수행.

---

## 6. 말 이동 규칙 (MOVE)

### 6.1 기본 이동

- 상하좌우 **직교 방향으로 1칸** 이동.
- 이동하려는 방향에 **벽이 가로막혀 있으면** 이동 불가.
- 보드 경계를 벗어날 수 없음.

### 6.2 점프 규칙 (상대 말과 인접 시)

- 이동하려는 방향에 **상대 말이 있으면**:
  - **6.2.1 직선 점프**: 상대 말 너머 칸이 비어 있고 벽이 없으면 → 상대를 뛰어넘어 그 칸으로 이동.
  - **6.2.2 대각선 점프**: 상대 말 너머에 **벽** 또는 **보드 끝**이 있으면 → 상대 말 기준으로 **좌/우 대각선 칸**으로 이동 가능 (각 대각선 칸으로 가는 길에 벽이 없어야 함).
- 점프는 자기 턴의 1회 이동에 포함됨 (추가 턴 없음).

### 6.3 이동 검증 의사코드

```pseudo
function getLegalMoves(state, player):
    moves = []
    for each direction in [N, S, E, W]:
        adj = player.position + direction
        if not onBoard(adj): continue
        if wallBetween(player.position, adj): continue
        if cellOccupied(adj):
            # 점프 시도
            beyond = adj + direction
            if onBoard(beyond) and not wallBetween(adj, beyond) and not cellOccupied(beyond):
                moves.add(beyond)
            else:
                # 대각선 점프
                for diag in perpendicular(direction):
                    diagCell = adj + diag
                    if onBoard(diagCell) and not wallBetween(adj, diagCell) and not cellOccupied(diagCell):
                        moves.add(diagCell)
        else:
            moves.add(adj)
    return moves
```

## 7. 벽 설치 규칙 (WALL)

벽 설치는 다음 **모든 조건**을 만족해야 유효:

1. **벽 개수 잔여**: 현재 플레이어의 `wallsRemaining > 0`.
2. **위치 유효성**: `col ∈ [0..7]`, `row ∈ [0..7]`.
3. **겹침 금지**: 동일 좌표/방향의 벽이 이미 있으면 안 됨.
4. **교차 금지**: 한 교차점에서 수평 벽과 수직 벽이 동시에 존재할 수 없음.
   - 예: `(c, r, "H")`가 있으면 `(c, r, "V")` 설치 불가, 그 반대도 동일.
5. **부분 겹침 금지**: 수평 벽 `(c, r, "H")`는 `(c-1, r, "H")` 및 `(c+1, r, "H")`와 겹침. 수직도 동일.
6. **경로 보존 (가장 중요)**:
   - 벽을 설치한 후에도 **모든 플레이어가 자신의 목표 줄에 도달 가능한 경로가 최소 1개 이상** 존재해야 함.
   - 검증: 가상으로 벽을 추가한 상태에서 각 플레이어 위치 → 목표 줄까지 **BFS/DFS**로 경로 탐색.
   - 한 명이라도 경로가 없으면 → 해당 벽 설치는 **불법**.

---

## 8. 승리 조건

- 자기 턴 종료 시 자신의 말이 **목표 줄(goal row/col)** 위 어느 칸에라도 있으면 즉시 승리.
- `status = "finished"`, `winner = currentPlayer.id`.

## 9. 권장 아키텍처 (웹 게임)

### 9.1 기술 스택 제안

- **프론트엔드**: React + TypeScript + Vite, 상태관리는 `useReducer` 또는 Zustand
- **렌더링**: SVG 또는 Canvas (셀 + 벽 슬롯 + 호버 미리보기)
- **백엔드(멀티플레이용)**: Node.js + WebSocket(`ws` 또는 Socket.IO)
- **로컬 우선**: 1단계로 동일 브라우저 2인 hotseat 모드 먼저 구현 → 2단계로 온라인 매칭

### 9.2 모듈 분리

```

    board.ts          # 보드/벽 좌표 유틸

    types.ts
  /ui
    Board.tsx         # 9x9 셀 + 벽 슬롯 렌더

  /net

```

## 10. UX 요구사항

1. **현재 턴 표시**: 어느 플레이어 차례인지 명확히 표시.
2. **남은 벽 개수**: 각 플레이어별로 항상 표시.
3. **합법 수 하이라이트**: 말 클릭 시 이동 가능한 칸 강조.
4. **벽 미리보기**: 벽 슬롯 호버 시 반투명으로 미리보기 + 불법 시 빨간색.
5. **되돌리기(Undo)**: `history`를 활용한 1회 되돌리기 (옵션).
6. **종료 화면**: 승자 표시 + 재시작 버튼.

## 11. 엣지 케이스 체크리스트

- [ ] 두 말이 마주보고 있고 그 너머에 벽이 있을 때, 양쪽 대각선이 모두 가능한지 검증.
- [ ] 대각선 점프 시, 대각선 경로 자체에 벽이 있는지도 검증.
- [ ] 벽 설치로 자기 자신의 경로도 막을 수 없음 (모든 플레이어 경로 보존).
- [ ] 보드 가장자리(0, 8행/열)에 벽 설치 시 좌표 범위 검증.
- [ ] 동시 두 벽 부분 겹침 (예: `(3,4,"H")`와 `(4,4,"H")`) 차단.
- [ ] 잔여 벽 0개일 때 벽 설치 버튼/모드 비활성화.

- **T5**: `(3, 4, "H")` 설치 후 `(3, 4, "V")` 시도 → 교차 금지로 거부.
