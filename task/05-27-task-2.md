# AI 상대 대전 기능 추가

## 목표
혼자서도 플레이할 수 있도록 AI 상대와 대전하는 방을 만든다.
사람 둘이 매칭되는 기존 방 흐름은 그대로 유지하고, "AI 상대" 옵션을 별도로 추가한다.

---

## 1. 동작 방식

### 1.1 방 만들기 흐름
- 로비에서 "AI 상대" 토글을 켜고 방 생성하면, p2가 자동으로 AI로 채워진다.
- 사람용 방처럼 "상대 기다리는 중" 화면을 거치지 않고, 즉시 게임 화면으로 진입한다.
- 5룰/6룰, 단판/3라운드 모드는 동일하게 선택할 수 있어야 한다.

### 1.2 AI 행동 시점
- AI 차례가 되면 클라이언트가 자동으로 `apply_action`을 호출한다.
- 즉시 두지 않고 1~2초 자연스러운 딜레이를 둔다 (사람처럼 보이게).
- "생각 중..." 같은 미세한 인디케이터를 상단바에 표시해도 좋다.

### 1.3 AI 실행 위치
- **클라이언트 휴리스틱**으로 구현한다. (AI 서버나 LLM 호출 없음)
- 단점: 현재 방을 보고 있는 인간 플레이어의 탭이 닫히면 AI도 멈춘다. → 게임 특성상 1인 플레이이므로 큰 문제 아님.
- 새로고침해도 게임 상태는 DB에 있으므로 복귀 가능. AI는 이어서 다시 자동 진행.

---

## 2. AI 휴리스틱 설계

### 2.1 플레이/버리기 단계
손패의 각 카드에 대해 가능한 행동마다 점수를 매기고 최고점을 선택한다.

**카드를 탐험에 두기 (PLAY) 점수 계산:**
- 해당 색 탐험에 이 카드를 더한 후의 예상 최종 점수 변화
- 베팅 카드는 이미 그 색에 베팅 ≥ 1장 또는 숫자 ≥ 4장 이상 둘 가망이 있을 때만 양수
- 같은 색에 더 큰 숫자 카드가 손에 많을수록 점수 가산
- 이미 시작한 탐험에 두는 것은 가산 (스타팅 비용 -20을 이미 지불했으므로)

**카드를 버리기 (DISCARD) 점수 계산:**
- 내 색이 아니면 약한 양수
- 상대가 시작한 색이면 페널티 (상대에게 줄 가능성)
- 미시작 색 중 손패에 카드가 1장만 있으면 가산 (놓을 가망 낮음)

### 2.2 드로우 단계
- 각 버림 더미 맨 위 카드:
  - 내가 즉시 둘 수 있고 가치 있으면 강한 양수 (그쪽에서 가져옴)
  - 상대가 그 색 탐험을 진행 중이고 큰 숫자라면 차단 목적의 약한 가산
- 그 외에는 덱에서 드로우 (정보 가치, 평균 카드)

### 2.3 베팅(협상카드) 판단
- 그 색에 이미 둔 베팅카드 개수 + 손패에 남은 같은 색 숫자 카드 합산이 충분히 클 때만 베팅
- 한 색에 최대 3장 베팅 가능하지만 보통 2장이 최적

### 2.4 6룰 목표카드 고려
- 미달성 목표 중 이번 행동으로 즉시 달성 가능한 목표 → 그 행동에 목표 점수만큼 가산
- 거의 다 왔으면 (예: "한 색에 3장" 목표가 있고 2장 둠) 그 색 진행에 가산

### 2.5 난이도 (선택)
- v1은 **단일 난이도 (보통)** 로 시작.
- 추후 쉬움/어려움 추가 시:
  - 쉬움: 합법 행동 중 무작위 선택
  - 어려움: 휴리스틱 + 손패 추적 + 더 신중한 베팅

## 3. DB 변경

### 3.1 컬럼 추가
`games` 테이블:
- `is_ai_p2 boolean not null default false` — p2가 AI인지 여부

### 3.2 신규 RPC: `create_ai_game(_p1_token, _mode, _ruleset)`
- `create_game`과 동일하게 방을 만들되 `is_ai_p2 = true`로 설정.
- 같은 트랜잭션 안에서 가짜 AI 토큰을 발급하여 p2 자리를 채운다 (`join_game` 로직 인라인).
- p2의 카드도 같이 분배되고 `turn_started_at = now()`로 시작.
- 반환: `{ room_code, role: 'p1' }`.

### 3.3 `get_state` 반환에 `is_ai_p2` 포함
- 클라이언트가 AI 방인지 알아야 자동 진행이 가능.

### 3.4 보안
- AI의 토큰은 DB에 저장은 하되 클라이언트에 노출되어도 된다 (어차피 1인 게임).
- 클라이언트가 `is_ai_p2 = true`인 방에 한해 AI 토큰을 가져와 p2로서 행동할 수 있게 한다.
- 대안: `apply_action`에 `_as_ai boolean` 플래그를 두어 토큰 검증을 우회하는 방식. (보안과 단순성 사이 트레이드오프)

권장: AI 토큰을 `get_state` 응답에 포함시키거나, 별도 RPC `get_ai_token(_room_code, _p1_token)`을 추가하여 인간 p1이 자신의 토큰으로 p2 AI 토큰을 받아오게 한다.

## 4. 프론트엔드 변경

### 4.1 `src/lib/types.ts`
- `GameState`에 `is_ai_p2: boolean` 추가
- `AiDifficulty` 타입 (장차 확장용, v1은 'normal' 고정)

### 4.2 `src/lib/api.ts`
- `createAiGame(mode, ruleset)` 함수 추가 → `create_ai_game` RPC 호출
- (필요 시) `applyActionAsAi(roomCode, aiToken, action)` 헬퍼

### 4.3 `src/lib/ai.ts` (신규)
- `chooseAction(state: GameState, hand: number[], aiRole: Role): Action`
  - 플레이/버리기 단계의 결정 + 드로우 단계의 결정 둘 다 처리
- 내부 헬퍼:
  - `evaluatePlay(cardId, state, hand, role)`
  - `evaluateDiscard(cardId, state, hand, role)`
  - `evaluateDraw(state, hand, role)` — 덱 vs 버림 더미 각각의 가치
  - `expeditionPotential(cardsAlready, restOfHandSameColor, ruleset)` — 그 색 탐험의 향후 가치 예측

### 4.4 `src/components/Lobby.tsx`
- 룰 토글 위 또는 아래에 "상대" 토글 추가:
  - `사람` (기본) — 기존 흐름
  - `AI` — `createAiGame` 호출
- AI 선택 시 안내 문구: "AI와 즉시 대전합니다"

### 4.5 `src/components/Game.tsx`
- `state.is_ai_p2`가 true일 때:
  - p2 자리는 `상대 기다리는 중` 화면 건너뛰기 (이미 join된 상태)
  - `상대 차례`일 때 1~2초 후 `ai.chooseAction()`을 호출하고, AI 토큰으로 `apply_action` 호출
  - "AI 생각 중..." 마이크로 인디케이터
- 라운드 시작 시에도 AI가 첫 플레이어면 자동으로 첫 액션 수행

### 4.6 종료/재시작
- 라운드 종료 화면에서 인간이 "다음 라운드" 버튼을 누르면 `next_round` 호출 → 새 라운드 시작 → AI가 첫 플레이어면 다시 자동 진행
- 매치 재시작도 마찬가지

## 5. UX 디테일

- AI는 결정이 즉시 가능하므로, 사람처럼 보이게 **랜덤 800~1800ms** 지연 후 행동
- 카드 사운드/애니메이션은 AI 턴에도 그대로 작동해야 함 (이미 상태 diff 기반으로 동작하므로 자동)
- 상단바에 AI 모드 표시 (예: `🤖 AI` 칩)
- 60초 타임아웃은 AI 턴에는 의미 없지만, 인간이 AI 응답을 기다리는 동안에도 인간의 손패가 무작위로 두어지면 안 되므로 — **인간 차례 타이머만 카운트**한다. (즉 `state.turn === human_role`일 때만 타이머 동작)
